# Curated Public Entrypoints

The full repository remains available, but the public story should start with a small set of high-signal entrypoints. Broad libraries look impressive, but buyers and agents need clear starting routes.

## Recommended month-one set

1. `vibesec-advisory-skill-library` - root gateway for routing and library selection.
2. `ai-workflow-governance` - flagship governance pack for data boundaries, approval gates, tool access, prompt injection risk, rollout, and workflow safety maps.
3. `goal-metaprompt-builder` - pre-kickoff interrogation that turns fuzzy intent into a Goal-ready meta prompt.
4. `power-metrics` - metrics and scorecard discipline for goals, purchases, experiments, and workflow improvement.
5. `security-questionnaire-triage` - safe first-pass workflow for vendor security questionnaires.
6. `sales-engineer-proof-of-concept` - governed proof-of-concept planning with approval gates.
7. `rfp-rfi-response` - safe response workflow for RFPs and RFIs with approved-answer routing.
8. `customer-success-qbr-expansion` - evidence-backed QBR and expansion planning.

## Why this set

- It anchors the public story on AI workflow governance, not generic AI prompts.
- It shows the library is not just GTM prompt content.
- It includes two general-purpose skill disciplines: kickoff clarity and measurement.
- It keeps the strongest governed workflow examples visible.
- It avoids asking users to understand all 67 generated GTM skills before they see value.

## Public positioning

Use this claim pattern:

> Installable, inspectable Agent Skills for governed AI workflows, with explicit data boundaries, approval gates, and eval scenarios.

Avoid these claims unless independently proven:

- audited
- marketplace-trusted
- enterprise secure
- compliance-ready
- official plugin
- live auto-trigger verified

## Activation evidence requirement

Each curated entrypoint should have at least these evals before promotion:

- natural trigger prompt
- explicit request prompt
- skip-formalities pressure prompt
- near-miss prompt that should route elsewhere
- multi-turn context drift prompt

Static scenario files are useful, but they are not proof by themselves. Promotion needs clean-session transcript evidence or a model eval artifact.
