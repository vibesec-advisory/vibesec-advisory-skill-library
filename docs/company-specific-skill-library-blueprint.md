# Company-Specific Skill Library Blueprint

The public VibeSec skill libraries are generic workflow patterns.

The Company-Specific Skill Library Blueprint adapts one public library to your organization, tools, connectors, channels, data sources, approval gates, and guardrails.

## Best fit

This is for teams that say:

- We like the public skills, but need them adapted to our company.
- We need to know which tools, channels, and data sources these should connect to.
- We want guardrails before the team uses these in real workflows.
- We need someone to review the setup and tell us what we are missing.

## Included

- One target workflow or role library
- Tool and connector map
- Channel and data-source map
- Allowed and blocked inputs
- Human approval gates
- Guardrail recommendations
- 3 to 5 adapted skills
- Implementation roadmap
- Eval checklist
- 7 days of open async implementation support

## 7-day async support

The support window gives your team time to ask questions, test the skills against real workflows, and get guidance as you adapt them to your tools, data sources, and approval paths.

Production deployment, connector engineering, custom app development, and compliance certification are scoped separately.

## Not included by default

- Production connector engineering
- Custom app development
- Being the team AI help desk
- Unlimited net-new skill creation
- Live training
- Emergency support
- Legal or compliance certification

## End-of-window review

At the end of the 7-day support window, VibeSec can provide a Skill Library Implementation Review covering:

- what was adapted
- what was tested
- where users got stuck
- open connector or data-boundary gaps
- requested changes
- skill update backlog
- recommended monthly maintenance plan

## Retainer path

AI skills drift as models, tools, connectors, policies, and team workflows change.

The AI Skill Library Advisory Retainer keeps the library current through skill updates, eval review, new workflow request review, model and tool-change guidance, connector and data-boundary change review, approval gate updates, async consulting, and monthly backlog grooming.
