# Metric Principles

## Good metrics are decision tools

A metric is useful only if it changes a decision, detects risk, or improves behavior.

## Metric families

- Outcome metric: the result that matters.
- Leading indicator: an earlier signal that predicts the outcome.
- Input metric: effort, volume, resources, or coverage.
- Process metric: cycle time, handoff quality, defect rate, or rework.
- Guardrail metric: the signal that prevents a bad local optimum.
- Counter-metric: a measure that would disprove the strategy.

## Quality checks

Score each candidate from 1 to 5:

- actionability
- controllability
- measurement latency
- data availability
- resistance to gaming
- stakeholder trust
- connection to business outcome

## Common tradeoffs

- Speed can beat quality when buyer patience or competitive timing decides the deal.
- Quality can beat speed when mistakes create legal, security, safety, or trust risk.
- Adoption can beat feature volume when the bottleneck is behavior change.
- Risk reduction can beat ROI when the buyer is avoiding downside.
- Learning rate can beat output volume in early experiments.

## Minimum metric contract

Every recommended metric should define:

- decision supported
- owner
- source of truth
- baseline
- target
- measurement window
- review cadence
- guardrail metric
- likely gaming behavior
- what action changes when the metric moves
